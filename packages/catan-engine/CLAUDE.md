# catan-engine library

## Layout

The package is organised into three subpackages by role, layered so imports only
ever point "down" (`env` → `mechanics` → `board`; no cycles):

- `board/` — **data**: the game's primitives (enums + constant tables) and its
  static/dynamic representation.
- `mechanics/` — **logic**: the traceable single-game rule helpers and the
  action layer that composes them.
- `env/` — **RL surface**: the batched env and the single-game PettingZoo-AEC
  wrapper.

## Source files

Keep this section up to date when adding, removing, or significantly changing a file.

### `board/` — primitives + representation

- `board/tile.py` — `Tile` enum: the six resource types (sheep, wheat, wood, brick, ore, desert).
- `board/port.py` — `Port` enum: the six port types (one per resource plus the 3:1 general port).
- `board/resources.py` — Resource constants (`N_PLAYERS`, `N_RESOURCES`, `BANK_INITIAL`), build-cost tuples (`ROAD_COST`, `SETTLEMENT_COST`, `CITY_COST`), jaxtyping aliases for resource arrays, `bank_stock()` (remaining stock of a single resource in one game), and `compute_bank_resources()` which derives remaining bank stock from player holdings.
- `board/dev_cards.py` — `DevCard` enum, deck counts (`DEV_CARD_COUNTS`), and `DEV_CARD_COST`. The dev-card *arrays* live on `BoardState` (see state.py).
- `board/layout.py` — Static board representation. Defines `BoardLayout` (tile resources, number tokens, port allocation), the module-level jaxtyping aliases for its arrays, and the static `int32` incidence maps generated once by `_generate_mappings()`: `EDGE_V` (edge→endpoints — the COO `edge_index`), `TILE_V` (tile→corner vertices), and `PORT_V` (port→vertices). These are dense and padding-free — the rule modules derive vertex-incidence on the fly by scattering over them (PyG-style; see `mechanics/placement` / `mechanics/trade` / `mechanics/dice`), so no ragged vertex→* reverse maps or padding sentinels are stored (shape invariants checked in `tests/board/test_layout.py`). `NO_INDEX` survives only as the uint8 "unclaimed award" marker on `BoardState`. `make_layout()` randomly generates a batched `BoardLayout`; `desert_tile()` reads the per-lane desert tile index off a `tile_resource` (the robber's rulebook starting position, needed because tile positions are shuffled). Also exposes host-side cube-coordinate ↔ index lookups for human-readable board inspection — `vertex_index`/`vertex_cube`, `edge_index`/`edge_cubes`, `tile_index`/`tile_cube` (cube `(q, r, s)` triples: tile centres sum to 0, vertices to ±1) — plain Python, not for use inside a trace; `tests/render.py` reuses these instead of re-deriving the geometry.
- `board/state.py` — Mutable game state. Defines `GamePhase`, building-stock/win constants, `BoardState` (board occupancy, dev-card deck/hand/knights, turn-flow flags such as phase/current_player/has_rolled/free_roads, and the Longest Road / Largest Army holders plus a PRNG key), `make_board_state()` which initialises a fresh game in the setup phase, `to_u8(x)` — the saturating uint8 cast through which state-array arithmetic is committed, and `tree_select(mask, a, b)` — the per-leaf `where` over two single-game states that the action layer uses for branchless application. Players are 0-indexed except in `vertex_owner`/`edge_road` (player + 1, 0 = empty). Also defines the **single-game (un-batched) jaxtyping aliases** the rule modules annotate with — `EdgeRoadVec`/`VertexOwnerVec`/`VertexTypeVec`/`PlayerMaskVec` plus the `IntScalar`/`BoolScalar`/`KeyScalar` scalars (the batch-free counterparts of the batched `*Array` aliases; `PortAllocVec`/`PlayerResourcesVec`/`DevDeckVec` live beside their batched forms in `layout`/`resources`/`dev_cards`).
- `board/__init__.py` — Top-level board type `Board = tuple[BoardLayout, BoardState]` (static layout paired with mutable state) plus action-agnostic construction/shaping helpers used by tests and callers (`make_board` — which seeds the robber on the desert tile (rulebook), `replicate`, `set_phase`, `to_main`, `give`, `place_settlement`/`place_road`/`place_city`, `give_dev_card`, `set_robber`). Re-exported at the package root, so `from catan_engine.board import Board, make_board` works.

### `mechanics/` — rule helpers + action layer

- **Rule modules** — the single-game, **traceable/vmappable** rule helpers, split by domain (all written without `int()`/NumPy/Python branching on traced values; vertex-incidence is computed by **scattering** per-edge/-tile values over the dense `EDGE_V` / `TILE_V` / `PORT_V` maps from `board/layout.py` — the JAX analogue of PyG message passing — so there are no padding sentinels; batch by wrapping callers in `jax.vmap`). Their public functions annotate array params with the **single-game** jaxtyping aliases above (`EdgeRoadVec`, `IntScalar`, ...), and these annotations are **runtime-enforced during tests**: `tests/conftest.py` installs a jaxtyping import hook (backed by `beartype`, a dev dependency) scoped to exactly these modules, so every call and `jit` trace checks shapes/dtypes. `mechanics/action` and `env` are excluded — their cores are annotated with the *batched* aliases but run unbatched under `vmap`, so enforcing them needs a batched→single-game re-annotation pass first.
  - `mechanics/placement.py` — settlement / road placement legality (`distance_rule_ok`, `settlement_connected`, `road_placeable`), via a `_scatter_to_vertices` helper over the edge_index.
  - `mechanics/awards.py` — Longest Road (the explicit-stack iterative DFS `longest_road_length` via `lax.while_loop`, edge-seeded; fed a sentinel-free CSR adjacency `_ADJ_INDPTR`/`_ADJ_EDGE`/`_ADJ_NBR` derived from `EDGE_V`) and Largest Army, plus the `recompute_*` award reassignment.
  - `mechanics/trade.py` — maritime `port_ratio` (lives here, not `board/port.py`, to avoid the `layout`→`port` import cycle).
  - `mechanics/dice.py` — `roll_dice` and the `distribute_resources` production payout (with the bank-cap rule).
  - `mechanics/robber.py` — `robber_victim_mask` and the random `steal`.
  - `mechanics/setup.py` — setup turn order (`SETUP_ORDER_ARR`, `N_SETUP`) and the 2nd-settlement `grant_setup_resources`.
  - `mechanics/development.py` — dev-card rules `playable_dev` / `draw_dev_card` (separate from `board/dev_cards.py` to avoid the `state`→`dev_cards` import cycle).
- `mechanics/action.py` — The engine's action layer. Defines `ActionResult` (SUCCESS/INVALID/GAME_COMPLETE), the `VecAction[Params]` base class, and **all 15** vectorized JAX-native actions (`SetupSettlement`, `SetupRoad`, `RollDice`, `Discard`, `MoveRobber`, `BuildRoad`, `BuildSettlement`, `BuildCity`, `BuyDevelopmentCard`, `PlayKnight`, `PlayRoadBuilding`, `PlayYearOfPlenty`, `PlayMonopoly`, `MaritimeTrade`, `EndTurn`; DomesticTrade is intentionally deferred). Also holds the single-game **economy helpers** these cores share — build-cost vectors (`ROAD_COST_ARR` etc.), `roads_left`, `count_settlements` / `count_cities`, `can_afford`, `pay`, and `player_total_vp` (generic uint8 saturation lives on `state.to_u8`, bank stock on `resources.bank_stock`). Each action's core is a single-game traceable transition; the public `is_available` / `__call__` are module-level `jit(vmap(...))`, so they run a whole batch at once (params are batched arrays — scalar params are `(batch,)`, multi-field params are tuples of `(batch,)` arrays — and the outcome is a `(batch,)` array of `ActionResult` codes). Application is branchless: the candidate state is always computed, then selected with `state.tree_select` against the `is_available` mask. Per-action tests live in `tests/mechanics/actions/` (pytest fixtures in `tests/mechanics/actions/conftest.py`; the shared ASCII `BoardRenderer` lives in `tests/render.py`).

**Engine path:** the engine is fully JAX-native (the `mechanics/` rule modules + `mechanics/action.py`); the old NumPy single-game path has been removed. The trusted differential oracle is the standalone **`catan-reference`** package (a plain-Python, rulebook-faithful implementation), bridged to the engine by `tests/conversion.py` — which converts a batched engine `Board` into a list of reference `Game`s (positions translated by cube coordinate) and exposes the per-rule wrappers that `tests/mechanics/test_rules.py` (and friends) check against. `tests/test_reference_equivalence.py` drives both engines with the same action stream and asserts full-state agreement across random games. None of this is part of the shipped package.

### `env/` — RL surface

- `env/batched.py` — RL environment core. Two layers: (1) the functional batched `step(board, action_type, params)` / `available(...)` — a single `(ActionType, ActionParams)` interface over all 15 actions, `jit(vmap(...))` over the `lax.switch` dispatchers in `mechanics/action.py`; (2) `BatchedCatanEnv`, a stateful **batched PettingZoo-AEC** env over that core (batch axis = parallel games; the acting agent in each lane is its `current_player`, or the next owing player during DISCARD). AEC surface adapted to batches: `agent_selection` is a `(B,)` array; `rewards`/`terminations`/`truncations` are `(B, N_PLAYERS)` arrays; `infos` is a single dict of batched arrays carrying `"action_mask"`. `observe(agent)` gives a **partial** view (own hand/dev cards in full, public counts only for opponents); `last()` returns the acting agents' slices. Reward is `"sparse"` (win/loss) or `"vp_delta"` (configurable); terminated lanes **auto-reset** to fresh boards by default (`auto_reset=False` freezes them instead — used by the single-game AEC wrapper in `env/aec.py`). `action_mask()` gives a `(B, N_ACTION_TYPES)` per-type legality sweep and `available_indices(action_type)` a `(B, D)` sweep over an index-parameterized action's primary domain. Spaces are described with dependency-free `Discrete`/`Box` descriptors. Re-exports `ActionType`, `ActionParams`, `ActionResult`, `N_ACTION_TYPES`.
- `env/__init__.py` — re-exports `env/batched.py`'s public API so `catan_engine.env.step`, `catan_engine.env.BatchedCatanEnv`, etc. resolve at the package root as before.
- `env/aec.py` — Single-game **PettingZoo-AEC** wrapper `CatanAECEnv` (and an `env()` constructor) around `BatchedCatanEnv(batch_size=1, auto_reset=False)`. Inherits `pettingzoo.utils.env.AECEnv`: turn-at-a-time `reset`/`step(action)`/`observe`/`last`, per-agent `rewards`/`terminations`/`truncations`/`infos` dicts, and gymnasium spaces. The action space is a single flat `Discrete` enumerating every concrete move (`_build_action_table` maps flat index → `(ActionType, idx, target)`; the lone DISCARD action's per-resource amounts are filled in canonically); legality is exposed PettingZoo-style as `observation["action_mask"]` (computed by querying `available` over the whole flat set on the replicated board). Observations are the partial view from `BatchedCatanEnv.observe`, wrapped `{"observation": {...}, "action_mask": ...}`. Game-over terminates all agents (then the base class dead-steps them out). Needs the optional `rl` extra (`pettingzoo`, `gymnasium`); validated by `tests/env/test_aec.py` (incl. PettingZoo's `api_test`).
