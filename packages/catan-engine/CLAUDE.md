# catan-engine library

## Layout

The package is organised into three subpackages by role, layered so imports only
ever point "down" (`env` → `mechanics` → `board`; no cycles):

- `board/` — **data**: the game's primitives (enums + constant tables) and its
  static/dynamic representation.
- `mechanics/` — **logic**: the traceable single-game rule helpers and the
  action layer that composes them.
- `env/` — **RL surface**: the batched env and the single-game PettingZoo-AEC
  wrapper.

## Source files

Keep this section up to date when adding, removing, or significantly changing a file.

### `board/` — primitives + representation

- `board/tile.py` — `Tile` enum: the six resource types (sheep, wheat, wood, brick, ore, desert).
- `board/port.py` — `Port` enum: the six port types (one per resource plus the 3:1 general port).
- `board/resources.py` — Resource constants (`N_PLAYERS`, `N_RESOURCES`, `BANK_INITIAL`), build-cost tuples (`ROAD_COST`, `SETTLEMENT_COST`, `CITY_COST`), jaxtyping aliases for resource arrays, `bank_stock()` (remaining stock of a single resource in one game), and `compute_bank_resources()` which derives remaining bank stock from player holdings.
- `board/dev_cards.py` — `DevCard` enum, deck counts (`DEV_CARD_COUNTS`), and `DEV_CARD_COST`. The dev-card *arrays* live on `BoardState` (see state.py).
- `board/layout.py` — Static board representation. Defines `BoardLayout` (tile resources, number tokens, port allocation), the module-level jaxtyping aliases for its arrays, and the static `int32` incidence maps generated once by `_generate_mappings()`: `EDGE_V` (edge→endpoints — the COO `edge_index`), `TILE_V` (tile→corner vertices), and `PORT_V` (port→vertices). These are dense and padding-free — the rule modules derive vertex-incidence on the fly by scattering over them (PyG-style; see `mechanics/placement` / `mechanics/trade` / `mechanics/dice`), so no ragged vertex→* reverse maps or padding sentinels are stored (shape invariants checked in `tests/board/test_layout.py`). `NO_INDEX` survives only as the uint8 "unclaimed award" marker on `BoardState`. `make_layout()` randomly generates a batched `BoardLayout`; `desert_tile()` reads the per-lane desert tile index off a `tile_resource` (the robber's rulebook starting position, needed because tile positions are shuffled). Also exposes host-side cube-coordinate ↔ index lookups for human-readable board inspection — `vertex_index`/`vertex_cube`, `edge_index`/`edge_cubes`, `tile_index`/`tile_cube` (cube `(q, r, s)` triples: tile centres sum to 0, vertices to ±1) — plain Python, not for use inside a trace; `tests/render.py` reuses these instead of re-deriving the geometry.
- `board/state.py` — Mutable game state. Defines `GamePhase`, building-stock/win constants, `BoardState` (board occupancy, dev-card deck/hand/knights, turn-flow flags such as phase/current_player/has_rolled/free_roads, and the Longest Road / Largest Army holders plus a PRNG key), `make_board_state()` which initialises a fresh game in the setup phase, `to_u8(x)` — the saturating uint8 cast through which state-array arithmetic is committed, and `tree_select(mask, a, b)` — the per-leaf `where` over two single-game states that the action layer uses for branchless application. Players are 0-indexed except in `vertex_owner`/`edge_road` (player + 1, 0 = empty). Also defines the **single-game (un-batched) jaxtyping aliases** the rule modules annotate with — `EdgeRoadVec`/`VertexOwnerVec`/`VertexTypeVec`/`PlayerMaskVec` plus the `IntScalar`/`BoolScalar`/`KeyScalar` scalars (the batch-free counterparts of the batched `*Array` aliases; `PortAllocVec`/`PlayerResourcesVec`/`DevDeckVec` live beside their batched forms in `layout`/`resources`/`dev_cards`).
- `board/__init__.py` — Top-level board type `Board = tuple[BoardLayout, BoardState]` (static layout paired with mutable state) plus action-agnostic construction/shaping helpers used by tests and callers (`make_board` — which seeds the robber on the desert tile (rulebook), `replicate`, `set_phase`, `to_main`, `give`, `place_settlement`/`place_road`/`place_city`, `give_dev_card`, `set_robber`). Re-exported at the package root, so `from catan_engine.board import Board, make_board` works.

### `mechanics/` — rule helpers + action layer

- **Rule modules** — the single-game, **traceable/vmappable** rule helpers, split by domain (all written without `int()`/NumPy/Python branching on traced values; vertex-incidence is computed by **scattering** per-edge/-tile values over the dense `EDGE_V` / `TILE_V` / `PORT_V` maps from `board/layout.py` — the JAX analogue of PyG message passing — so there are no padding sentinels; batch by wrapping callers in `jax.vmap`). Their public functions annotate array params with the **single-game** jaxtyping aliases above (`EdgeRoadVec`, `IntScalar`, ...), and these annotations are **runtime-enforced during tests**: `tests/conftest.py` installs a jaxtyping import hook (backed by `beartype`, a dev dependency) scoped to exactly these modules, so every call and `jit` trace checks shapes/dtypes. `mechanics/common`, `mechanics/turn`, `mechanics/action`, and `env` are excluded — they hold functions annotated with the *batched* aliases that run unbatched under `vmap` (the cores in the hooked rule modules, by contrast, are annotated single-game with `IntScalar` / `BoolScalar` / `SingleResources`, so they *are* enforced).
  Each topical module holds its sub-rule helpers **and** the action cores built on them (a core is a single-game traceable transition; each exposes the *batched* public API `<name>_available(board, params)` / `<name>_step(board, params)` — module-level `jit(vmap(...))` — plus the private single-game `_<name>_avail` / `_<name>_apply` that the dispatch in `action.py` calls). A core applies **only** its core state change and returns `SUCCESS`/`INVALID`; the award reassignment and the win check are **stage 2**, run once per step by `awards.resolve_step` (see the `awards.py` and `action.py` bullets). The few `*_step` wrappers whose action can change an award or win the game (BuildRoad / BuildSettlement / BuildCity / BuyDevelopmentCard / PlayKnight) route their core output through the batched `awards.resolve_step_b` so the standalone API stays fully resolved.
  - `mechanics/placement.py` — settlement / road placement legality (`distance_rule_ok`, `settlement_connected`, `road_placeable`, `setup_road_placeable`, via a `_scatter_to_vertices` helper over the edge_index) **and** the `BuildRoad` / `BuildSettlement` / `BuildCity` cores.
  - `mechanics/awards.py` — Longest Road (the explicit-stack iterative DFS `longest_road_length` via `lax.while_loop`, edge-seeded; fed a sentinel-free CSR adjacency `_ADJ_INDPTR`/`_ADJ_EDGE`/`_ADJ_NBR` derived from `EDGE_V`) and Largest Army, plus the `recompute_*` award reassignment. Each DFS frame carries its used-edge set as a **packed int32 bitmask**: a running count over the player's edges (`local_of`) maps the `<= MAX_ROADS` (15) owned edges to bit positions, so set/test are scalar bit ops and the stack mask is `(STACK_CAP,) int32` rather than `(STACK_CAP, N_EDGES) bool` (~9x faster batch-100 rollout). Also the **stage-2 step resolver** `resolve_step(state, result)` (and its batched `resolve_step_b`): `recompute_awards` (both holders) then upgrade a `SUCCESS` to `GAME_COMPLETE` if any player hit the win threshold. Factored out of the cores so the expensive Longest Road DFS runs **once** per step, not once per `lax.switch` branch (every branch executes under `vmap`).
  - `mechanics/trade.py` — maritime `port_ratio` (lives here, not `board/port.py`, to avoid the `layout`→`port` import cycle) **and** the `MaritimeTrade` core.
  - `mechanics/dice.py` — `roll_dice` and the `distribute_resources` production payout (with the bank-cap rule) **and** the `RollDice` core.
  - `mechanics/robber.py` — the 7-resolution mechanics: `robber_victim_mask` and the random `steal`, the shared `valid_robber_victim` / `apply_steal` (reused by `PlayKnight`), **and** the `MoveRobber` / `Discard` cores.
  - `mechanics/setup.py` — setup turn order (`SETUP_ORDER_ARR`, `N_SETUP`) and the 2nd-settlement `grant_setup_resources` **and** the `SetupSettlement` / `SetupRoad` cores.
  - `mechanics/development.py` — dev-card rules `playable_dev` / `draw_dev_card` (separate from `board/dev_cards.py` to avoid the `state`→`dev_cards` import cycle) **and** the `BuyDevelopmentCard` / `PlayMonopoly` / `PlayYearOfPlenty` / `PlayRoadBuilding` / `PlayKnight` cores (Knight composes `robber`; its Largest Army recompute is deferred to the stage-2 `awards.resolve_step`).
  - `mechanics/turn.py` — the `EndTurn` core.
- `mechanics/common.py` — the shared vocabulary every core needs (so the rule modules stay leaves importing only `board.*` + `common`, with no cycle through `action`): `ActionResult` (SUCCESS/INVALID/GAME_COMPLETE) and those scalar codes, the MAIN-phase predicates (`main_after_roll` / `main_no_dev_played`), the **economy helpers** (build-cost vectors `ROAD_COST_ARR` etc., `roads_left`, `count_settlements` / `count_cities`, `can_afford`, `pay`, `player_total_vp`), `agent_selection_single`, and the action-layer jaxtyping aliases — batched (`Mask`, `ResultCode`, `IndexParam`, `TwoIndexParams`, `DiscardParams`) and single-game (`SingleResources`; cores also use `IntScalar` / `BoolScalar` from `board.state`). (Generic uint8 saturation lives on `state.to_u8`, bank stock on `resources.bank_stock`.)
- `mechanics/action.py` — The aggregation / enumeration layer on top of the cores. (1) The unified dispatch: `ActionType` (the 15-action enum; DomesticTrade is intentionally deferred), the packed `ActionParams`, and `apply_action` / `action_available` — a single `(action_type, params)` interface, `lax.switch` over the cores so it stays traceable/vmappable. Application is branchless (candidate always computed, then `state.tree_select`-ed against the legality mask) and **two-stage**: the switch applies the chosen action's core state change, then `awards.resolve_step` recomputes the awards and resolves the win **once** per step (factored out of the per-branch cores for the perf reason given in the `awards.py` bullet). (2) The **flat action table** (`_build_action_table` → `_ATYPE`/`_IDX`/`_TARGET`/`_DISCARD_FLAT`, plus the host-side `_canonical_discard`): one enumerated index per concrete move. (3) The switch-free **flat legality** (`_flat_available_b`) and the per-action-type / per-index legality enumerations (`_action_type_mask_b`, `_INDEX_MASKS`), which call each core directly over its own static slice of the parameter space rather than routing the table through the `lax.switch` (which `vmap` would lower to evaluating all 15 branches per entry). Re-exports `ActionResult` / `Mask` / `ResultCode` / `player_total_vp` from `common`. Per-action tests live in `tests/mechanics/actions/` (they call the functional `<name>_step` / `<name>_available`; pytest fixtures in `tests/mechanics/actions/conftest.py`; the shared ASCII `BoardRenderer` lives in `tests/render.py`).

**Engine path:** the engine is fully JAX-native (the `mechanics/` rule modules + `mechanics/action.py`); the old NumPy single-game path has been removed. The trusted differential oracle is the standalone **`catan-reference`** package (a plain-Python, rulebook-faithful implementation), bridged to the engine by `tests/conversion.py` — which converts a batched engine `Board` into a list of reference `Game`s (positions translated by cube coordinate) and exposes the per-rule wrappers that `tests/mechanics/test_rules.py` (and friends) check against. `tests/test_reference_equivalence.py` drives both engines with the same action stream and asserts full-state agreement across random games. None of this is part of the shipped package.

### `env/` — RL surface

- `env/batched.py` — RL environment core. Two layers: (1) the functional batched `step(board, action_type, params)` / `available(...)` — a single `(ActionType, ActionParams)` interface over all 15 actions, `jit(vmap(...))` over the `lax.switch` dispatchers in `mechanics/action.py`; (2) `BatchedCatanEnv`, a stateful **batched PettingZoo-AEC** env over that core (batch axis = parallel games; the acting agent in each lane is its `current_player`, or the next owing player during DISCARD). AEC surface adapted to batches: `agent_selection` is a `(B,)` array; `rewards`/`terminations`/`truncations` are `(B, N_PLAYERS)` arrays; `infos` is a single dict of batched arrays carrying `"action_mask"`. `observe(agent)` gives a **partial** view (own hand/dev cards in full, public counts only for opponents); `last()` returns the acting agents' slices. Reward is `"sparse"` (win/loss) or `"vp_delta"` (configurable); terminated lanes **auto-reset** to fresh boards by default (`auto_reset=False` freezes them instead — used by the single-game AEC wrapper in `env/aec.py`). `action_mask()` and `available_indices(action_type)` delegate to the `_action_type_mask_b` (`(B, N_ACTION_TYPES)` per-type sweep) and `_INDEX_MASKS` (`(B, D)` sweep over an index-parameterized action's primary domain) enumerations in `mechanics/action.py`. `random_actions(key)` samples a uniformly-random *legal* action per lane (the flat legality sweep `_flat_available_b`, then a per-lane masked argmax) — the random-rollout driver. The **flat action table** and switch-free **flat legality** both live in `mechanics/action.py` now (see that bullet); this module imports them. Spaces are described with dependency-free `Discrete`/`Box` descriptors. Re-exports `ActionType`, `ActionParams`, `ActionResult`, `N_ACTION_TYPES`.
- `env/__init__.py` — re-exports `env/batched.py`'s public API so `catan_engine.env.step`, `catan_engine.env.BatchedCatanEnv`, etc. resolve at the package root as before.
- `env/aec.py` — Single-game **PettingZoo-AEC** wrapper `CatanAECEnv` (and an `env()` constructor) around `BatchedCatanEnv(batch_size=1, auto_reset=False)`. Inherits `pettingzoo.utils.env.AECEnv`: turn-at-a-time `reset`/`step(action)`/`observe`/`last`, per-agent `rewards`/`terminations`/`truncations`/`infos` dicts, and gymnasium spaces. The action space is a single flat `Discrete` enumerating every concrete move (the flat action table — `_build_action_table` / `_canonical_discard` — lives in `mechanics/action.py` and is imported here; flat index → `(ActionType, idx, target)`, the lone DISCARD action's per-resource amounts filled in canonically); legality is exposed PettingZoo-style as `observation["action_mask"]` (the switch-free `_flat_available_b` flat sweep from `mechanics/action.py`, shared with `random_actions`). Observations are the partial view from `BatchedCatanEnv.observe`, wrapped `{"observation": {...}, "action_mask": ...}`. Game-over terminates all agents (then the base class dead-steps them out). Needs the optional `rl` extra (`pettingzoo`, `gymnasium`); validated by `tests/env/test_aec.py` (incl. PettingZoo's `api_test`).
